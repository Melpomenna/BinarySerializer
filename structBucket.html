Detailed Description

Контейнер для узлов в хеш-таблице

Внутренняя структура для хранения массива элементов

Bucket реализует динамический массив узлов для разрешения коллизий методом цепочек (separate chaining).

Стратегия роста:
  • Начальная емкость: 1 элемент
  • Рост: удвоение при заполнении (capacity *= 2)
  • Сжатие: не выполняется автоматически
Сложность операций:
  • Вставка (без коллизий): O(1) амортизированная
  • Поиск: O(n), где n = nodesCount
  • Удаление: O(n)
Warning
После clearBucket() необходимо повторно инициализировать bucket
See also
Node, insertIntoBucket(), clearBucket()

Непрозрачный тип, детали реализации скрыты в .c файле. Представляет собой бакет хеш-таблицы, содержащий массив элементов из типа Node
